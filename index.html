A website built with the purpose of implementing a conference management system for every party included (e.g. author, reviewer, chair). TYPESCRIPT ANDROID GCP C++ NGINX
Top 3 winner for Google Solution Challenge 2023. A tool to teach braille to visually impaired people. This project is composed of both hardware and software, all of it custom-made by our team. With this device, a teacher could help a student learn braille using the interface of their phone, where the student would then respond using the device.

One of the project with the most changes in its lifespan. Originally, it started as a hardware to interface with the internet using braille. It then changed to a web-based teaching device, ending up on an offline Android application.
The reason for these changes is due to our team's insistence on fulfilling a real-world need using the application. We repeatedly tested this on the local visually impaired organization, making sure we were on the right track.

Overall, the most fulfilling project that I have ever done, with the most amount of repositories.

GOLANG GRPC KUBERNETES
3-month internship and my first experience working in a large codebase. For more information regarding the company, click here. I was brought on to the company to tackle two objectives. One, try to learn as much as possible and two, start the effort in migrating the architecture to a distributed one. At that time, the codebase was monolithic and was getting somewhat bloated (some files reaching 23k LOC). To alleviate this, I was tasked with isolating one of the smaller module into a microservice and a decision was made to extract email functionality from the main server.

While it may seem simple, the refactoring effort to implement that is not trivial. The process of sending emails were chaotic and non-uniform, with files implementing their own email functions and templates. I first started by refactoring all of these functions into one single file and continue from there. The large codebase and my lacking knowledge in an actual working environment proved this to not be an easy task, but here's what I managed to achieve at the end of my internship:

  • Contained email functionalities into it's own repository, complete with CI/CD and build scripts adherent to company convention.
  • Deployed Kubernetes cluster with Kong API gateway and monitoring.
  • Implemented gRPC and protobuf contracts for the microservice.

    • Overall I am quite proud with what I have achieved in this internship. While the duration itself was quite short, the team's willingness to trust me with important resources allowed me to grow as a developer. I consider this a significant learning experience for my journey as a developer.
    Translate-inator
    link
    TYPESCRIPT ANGULAR GCP DOCKER NGINX
    Full-stack project of a note taking web application with a translation function. Deployed on a Google Compute Engine using Docker compose and nginx reverse proxy. This is a solo project that uses Google's Cloud Translation API. The backend is created using Typescript and Node.js, while the frontend uses Angular with rxjs. Deployment is handled with Github Action CI/CD.

    This project posed a different type of challenge in how I approached my development. I went into it with the mindset of "This should be maintainable by a complete stranger", which proved to be more difficult than I thought. Coding the functionality itself was not a problem, but everything surrounding it had to be easy to modify and should allow customization for different environments (production, development, etc.).

    The final result of this 13-day sprint is a full-stack application that can be run with two docker commands. 

    docker compose -f ./docker-compose.setup.yaml up -d
    docker compose up -d
